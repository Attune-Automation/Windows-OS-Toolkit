The following steps will guide you through the manual process.

Step 1 - WIN STD Deploy Hosts File

Deploy the specified hosts file to the new Windows server.

Connect via RDP:

mstsc /admin /v:{targetserverwin}

Login as user {windowsadministrator} and open a command prompt.

Root folder=/, Deploys to /C$/

Copy the File(s) PON_Win_Server_Hosts_File.tar to the target node.

/c$/

Step 2 - WIN STD Update Hostname

Set the hostname to the specified hostname input value.

Execute the following script: 

                
(Get-WmiObject Win32_ComputerSystem).Rename("{targetServerWin.hostname}")

Step 3 - WIN STD Set Date and Time Formats

Execute the following script: 

                
$culture = (Get-Culture).Clone()

$culture.DateTimeFormat.ShortTimePattern = 'HH:mm:ss'
$culture.DateTimeFormat.LongTimePattern = 'HH:mm:ss'

$culture.DateTimeFormat.ShortDatePattern = 'dd/MM/yyyy'
$culture.DateTimeFormat.LongDatePattern = 'dd MMMM yyyy'

Set-Culture $culture

Step 4 - WIN STD Set Timezone

Set the time zone to the specified time zone input value.

Execute the following script: 

                
Get-TimeZone '{targetServerWindowsTimezone}' | Set-TimeZone

Step 5 - WIN STD Set Country

Display the currently selected location, then set this to the specified Geo ID input value and display the new value.

Execute the following script: 

                
Write-Host "Country is currently:"
Get-WinHomeLocation

Write-Host "Setting the Country"
Write-Host "Settings -> Time & Language -> Region and Language -> Country or Region"
Set-WinHomeLocation -GeoID {targetServerWinGeoid}

Write-Host "Country has been changed to:"
Get-WinHomeLocation

Step 6 - WIN STD Set Language

Display the currently selected language, then set this to the specified OS language input value and display the new settings.

Execute the following script: 

                
Write-Host "Language list was:"
Get-WinUserLanguageList

Write-Host "Setting the Country"
Write-Host "Settings -> Time & Language -> Region and Language -> Languages"

$newLang = New-WinUserLanguageList -Language {targetServerWinLang}
Set-WinUserLanguageList -LanguageList $newLang -Force

Write-Host "Language list has been changed to:"
Get-WinUserLanguageList

Step 7 - WIN STD Set Language In Registry

Sometimes, the locale settings are not detected correctly or automatically updated when the location is updated. This script ensures that these are set to the correct values by updating the registry entries for the locale and locale name.

https://www.pofcommunity.com/articles?id=104

Execute the following script: 

                
$path_hkcu = 'HKCU:\Control Panel\International'
$path_hku = 'Microsoft.PowerShell.Core\Registry::HKEY_USERS\.Default\Control Panel\International'



$value = '{targetServerWinLangId}'
$name = 'Locale'
Set-Itemproperty -Path $path_hkcu -Name $name -Value $value
Set-Itemproperty -Path $path_hku -Name $name -Value $value

$value = '{targetServerWinLang}'
$name = 'LocaleName'
Set-Itemproperty -Path $path_hkcu -Name $name -Value $value
Set-Itemproperty -Path $path_hku -Name $name -Value $value

Step 8 - WIN STD Set Time Sync

Set the NTP server(s) to the specified list of NTP servers for synchronisation of system time and date.

Execute the following script: 

                
$path = "HKLM:\system\CurrentControlSet\Services\W32Time\Parameters"
$name = "NtpServer"

$value = Get-ItemProperty -Path $path -Name $name

Write-Output "Existing NtpServer Value: "$value.NtpServer

# JJC - I don't know if you CAN put more than one server in here.
$newValue = "{ntpServers.serverFqns},0x9"
Write-Output "Time Sync Servers: $newValue"

Write-Output "New NtpServer Value: "$newValue

Set-ItemProperty -Path $path -Name $name -Value $newValue

# ------------------------------------------------------------------------------
# Set the servers displayed in Control Panel -> Date and Time -> Internet Time (tab)

$path = "HKLM:\SOFTWARE\Microsoft\Windows\CurrentVersion\DateTime\Servers"

$fqns = "{ntpServers.serverFqns}" -Split ','
for ( $index = 0; $index -lt $fqns.count; $index++) {
    $fqn = $fqns[$index]
    Write-Output "Setting [$($index + 1)] : $fqn"
    Set-ItemProperty -Path $path -Name ($index + 1) -Value $fqn
}

Step 9 - WIN STD Set Browser HomePage

Sets the start page, search page, default page shown on launch, and default search engine for Internet Explorer to the specified Browser Home Page URL.

Execute the following script: 

                
$path_hklm = 'HKLM:\Software\Microsoft\Internet Explorer\Main'
$path_hklm32 = 'HKLM:\SOFTWARE\WOW6432Node\Microsoft\Internet Explorer\Main'
$path_hkcu = 'HKCU:\Software\Microsoft\Internet Explorer\Main'
$path_hku = 'Microsoft.PowerShell.Core\Registry::HKEY_USERS\.Default\Software\Microsoft\Internet Explorer\Main'

Set-Location 'Registry::\HKEY_USERS'


if ( -Not (Test-Path '.Default\Software\Microsoft\Internet Explorer')) {
    New-Item -Path '.Default\Software\Microsoft' -Name 'Internet Explorer'
}


if ( -Not (Test-Path '.Default\Software\Microsoft\Internet Explorer\Main')) {
    New-Item -Path '.Default\Software\Microsoft\Internet Explorer' -Name Main
}


$value = '{targetServerBrowserHomePage}'

$name = 'Start Page'
Set-Itemproperty -Path $path_hkcu -Name $name -Value $value
Set-Itemproperty -Path $path_hku -Name $name -Value $value
Set-Itemproperty -Path $path_hklm -Name $name -Value $value
Set-Itemproperty -Path $path_hklm32 -Name $name -Value $value

$name = 'Search Page'
Set-Itemproperty -Path $path_hkcu -Name $name -Value $value
Set-Itemproperty -Path $path_hku -Name $name -Value $value
Set-Itemproperty -Path $path_hklm -Name $name -Value $value
Set-Itemproperty -Path $path_hklm32 -Name $name -Value $value

$name = 'Default_Page_URL'
Set-Itemproperty -Path $path_hklm -Name $name -Value $value
Set-Itemproperty -Path $path_hklm32 -Name $name -Value $value

$name = 'Default_Search_URL'
Set-Itemproperty -Path $path_hklm -Name $name -Value $value
Set-Itemproperty -Path $path_hklm32 -Name $name -Value $value

Step 10 - WIN STD Disable Server Manager

If the target server is a Windows Server or Windows 10 instance, disable Server Manager from running at startup.

Execute the following script: 

                
# This may get run on Win10, It should just silently do nothing

if (Test-Path "HKCU:\Software\Microsoft\ServerManager") {
    Write-Host "Disabling ServerManager opening at login"
    New-ItemProperty -Path HKCU:\Software\Microsoft\ServerManager `
        -Name DoNotOpenServerManagerAtLogon `
        -PropertyType DWORD `
        -Value "0x1" -Force
} else {
    Write-Host "This is not a windows server"
}
